TLS Cipher Suite Registry

- TLS Cipher Suite are often mentioned in audit report

- but what's that mean ?

Cryptographic functions


  1. Key exchange: used to exchange the master secret key

  2. Authentication: used to authenticate the server

  3. Encryption: used to encrypt data

  4. Message Authentication Code: used to sign data

- TLS protocol used 4 cryptographic functions

Key exchange


  • DH (Diffie Hellman)
  • RSA
  • DHE (Diffie Hellman Ephemeral)
  • ECDH (Elliptic curve Diffie–Hellman) with P=256bits
  • ECDHE (Elliptic curve Diffie–Hellman Ephemeral) with P=256bits


/!\ Perfect Forward Secrecy

Authentication


  • DSS
  • RSA with key length 2048 bits
  • ECDSA with P=256bits

Encryption


  • DES
  • 3DES
  • RC4
  • AES CBC (128 or 256 bits)
  • AES GCM (128 or 256 bits)
  • ChaCha20 (256 bits)


/!\ Authenticated Encryption with Associated Data

Message Authentication Code


  • MD5
  • SHA1 (160bits)
  • SHA2 (256bits)
  • SHA3
