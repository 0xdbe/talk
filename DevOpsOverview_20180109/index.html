Three ways

1 - System Thinking

  • Prendre en compte le flux de travail entre les dev et la prod
  • Etudier le systeme complet pour identifier les les goulots d'étranglement (TOC : Theory des contraintes)
  • Intégration dans Agile
    • Inclure les Ops dans le spring planning meeting et les daily meeting
    • Inclure les taches des Ops dans le sprint backlog
    • Inclure la phase Deploy dans le Work in progress
  • Définir des objectifs communs et pas des objectifs locals (% de Fonctionnalité développé, bug trouvé / bug corrigé, % de dispo, ...)

2 - Feedback loop

  • Permettre aux Ops de faire un retour aux Dev.
  • Les dev doivent avoir accès au Telemetry dashboard et Log
  • Inclure les Ops dans les rétrospectives

Three Ways

3 - Culture of continual Experimentation and Learning

  • Mettre en place un system permettant aux personnes d'apprendre et de tester, ...
  • Sujet d'étude intégré dans le sprint backlog et restituer lors du sprint retrospective
