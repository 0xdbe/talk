Waterfall

The “traditional” approach


linear and sequential (non-iterative)

- Le processus traditionellement déployé dans les entreprises est le processus Waterfall

- Waterfall est un processus séquentiel et linéaire

- Problématique: taux d'insatisfaction important car le client ne vera le résultant qu'à la fin du processus.

Agile

The iterative approach

  • Manifesto for Agile Software Development (February 2001)

  • Product oriented

  • Cross functional team:
    • Small team (Two-pizza team)
    • Self-organizing and self-managing

  • Popular framework: Scrum and Kanban

- Changement de paradigme: projet vers produit

- Product approach: Avec les méthodes traditionelles, les personnes pensent travailler sur un projet temporaire et que ce sera terminé dans 5 à 6 mois et qu'ils passeront ensuite à un autre projet. La méthode agile utilise une approche produit qui doit être opérationnel pendant de nombreuses années.

- Cross functional team: l'équipe est consituée d'un scrum master, product owner et de développeur. La taille de l'équipe est une dizaine de personnes (two pizza team). Cette équipes est autonome.

Scrum sprint lifecycle

- Security tasks in the product backlog

- Security acceptance criteria in Definition of Done

Agile Security

  • Secure Scrum

  • Product backlog:
    • Security-focused stories (Safe Code)
    • Evil user stories (OWASP) or Abuser stories

  • Sprint backlog: security features will need to be included as applicable for every single sprint backlog feature.

  • Definition of Done: include secure acceptance criteria (Source code audit, ...).

- Security: secure scrum propose de traiter la sécurité par ajout de task dans le product backlog et ajouter un recette sécurité dans le Defition Of Done.

Example of user stories


  • As an employee, I can search for other employees by their last name.

  • As an architect/developer I want to ensure AND as QA I want to verify the user is protected by robust authentication and session management.

  • As a hacker, I can send bad data in the content of requests, so I can access data and functions for which I'm not authorized.
